Mastra Orchestrator Overview

The Mastra orchestrator is the GitHub-facing Code Assist service under apps/mastra-orchestrator/. It receives GitHub events, maintains Code Assist state, starts Codex sessions, runs deterministic tests, opens draft PRs, and posts comments/checks through a GitHub App.

Registered Workflows​

WorkflowPurpose
code-assist-issue-planningConvert GitHub issues into implementation-ready plans.
code-assist-issue-implementationImplement approved issue plans in bot-owned worktrees and draft PRs.
code-assist-pr-followupContinue implementation from PR comments, reviews, failed checks, close events, and promotion events.
code-assist-pr-testingRun reusable deterministic testing commands against a repo/worktree.
code-assist-test-triageUse read-only Codex to interpret failed tests and propose implementation fixes.

API Routes​

RouteMeaning
GET /code-assist/healthHealth/configuration check.
POST /code-assist/github/webhookGitHub webhook entry point.

Orchestrator Boundary​

Mastra coordinates. It does not replace X4 or Codex:

  • GitHub events and workflow state are Mastra concerns.
  • Worktree setup, git operations, sandbox startup, and deterministic tests are service concerns.
  • Agentic reasoning and edits are delegated to Codex through a structured harness.
  • PR evidence is posted only after deterministic execution or structured workflow state.
